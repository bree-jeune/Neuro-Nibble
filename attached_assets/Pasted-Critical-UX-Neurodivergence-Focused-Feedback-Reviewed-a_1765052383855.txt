Critical UX & Neurodivergence-Focused Feedback
Reviewed as: UX Designer + ADHD/Neurodivergence Subject Matter Expert

Overall Assessment: This is 80% excellent with 5 critical friction points that will cause abandonment. You've nailed the voice and core concept, but there are neurodivergent-specific interaction patterns being violated.

⚠️ CRITICAL ISSUES (Fix Before Launch)
1. Cognitive Load Crisis: "Break It Down" Has Too Many Fields
Problem (Screen 2 - "Break It Down" modal):

text
You're asking users to:
1. Name the task
2. Break into bites
3. Write first step
4. Estimate time
5. Choose energy level (Low/Medium/High)
6. Add more steps
That's 6 decisions WHILE they're already paralyzed.

Neurodivergent Issue:

Decision fatigue kills momentum

Too many empty fields = "I'm doing this wrong" anxiety

Energy selection BEFORE they've even started = premature optimization

UX Violation:
Progressive disclosure failure - you're showing everything at once.

Solution: Two-Step Flow

Step 1: Ultra-Minimal Entry

text
┌─────────────────────────────────────┐
│ Break It Down                       │
├─────────────────────────────────────┤
│                                     │
│ What's freezing you?                │
│ ┌─────────────────────────────────┐ │
│ │ grocery shopping                │ │
│ └─────────────────────────────────┘ │
│                                     │
│ What's the tiniest first step?      │
│ ┌─────────────────────────────────┐ │
│ │                                 │ │
│ └─────────────────────────────────┘ │
│                                     │
│          [Start This Bite]          │
│                                     │
│ "You can add more steps after.      │
│  Just start with one."              │
└─────────────────────────────────────┘
Step 2: AFTER They Start (Not Before)
Then offer:

Add another step?

Estimate time? (optional)

Match energy level? (optional)

Why This Works:

One step = no analysis paralysis

They get momentum FIRST

Optional complexity comes AFTER initial success

2. "0 of 2 bites complete" = Shame Trigger
Problem (Screen 1 - Task detail view):

The fraction counter "0 of 2 bites complete" creates:

Visible failure state (0 = you haven't started)

Pressure (2 bites = you committed to something)

Shame (the number judges you)

Neurodivergent Issue:
ADHD brains are hypersensitive to perceived failure. Seeing "0 complete" before you've even tried = discouragement.

UX Violation:
Showing incomplete state before user has chance to succeed.

Solution: Hide Progress Until First Action

Before ANY bites are done:

text
┌─────────────────────────────────────┐
│ grocery shopping                    │
├─────────────────────────────────────┤
│                                     │
│ Current bite:                       │
│ making a list                       │
│                                     │
│ [Mark Done]                         │
│                                     │
│ "You can stop after this step.      │
│  That still counts."                │
└─────────────────────────────────────┘
AFTER first bite is done:

text
┌─────────────────────────────────────┐
│ grocery shopping                    │
│ ✓ 1 bite done today                 │
├─────────────────────────────────────┤
│ Next bite:                          │
│ driving to the store                │
│                                     │
│ [Keep Going] [Stop Here]            │
└─────────────────────────────────────┘
Key Difference:

No fraction (no "0 of X" shame)

Shows what you DID, not what you didn't

Celebrates first action immediately

3. Energy Check-In Timing is Wrong
Problem (Screen 7 - Home screen with energy selector):

Energy selection happens BEFORE seeing tasks. This forces users to:

Abstract self-assessment ("How's my energy?")

Without context (energy for WHAT?)

Then match to a task

Neurodivergent Issue:
ADHD brains struggle with:

Interoception (knowing internal states without external cues)

Abstract energy assessment

Decision-making without concrete options

This is why users ignore the energy selector and jump straight to tasks.

Solution: Context-First, Then Energy

New Flow:

Step 1: Task Selection First

text
Home Screen:
┌─────────────────────────────────────┐
│ Saturday                            │
├─────────────────────────────────────┤
│ Pick up where you left off          │
│                                     │
│ [clear phone storage]  0/4 bites    │
│                                     │
│ Or start a new bite                 │
│ [+ New Task]                        │
└─────────────────────────────────────┘
Step 2: Energy Check AFTER Task Selected

text
┌─────────────────────────────────────┐
│ clear phone storage                 │
├─────────────────────────────────────┤
│ How's your energy for THIS task?    │
│                                     │
│ [Low]  [Medium]  [High]             │
│                                     │
│ "This helps me show the right       │
│  first step for you."               │
└─────────────────────────────────────┘
Why This Works:

Concrete question ("energy for THIS") vs. abstract ("energy today")

Users understand WHY they're answering

Energy affects what they see next (Low = smaller first step)

4. "You Can Stop After This Step" is Buried
Problem (Screen 1):

The permission statement "You can stop after this step. That still counts." is:

At the BOTTOM (below the fold on small screens)

In italic gray text (deprioritized visually)

Easy to miss

Neurodivergent Issue:
This is your most important message. ADHD users need:

Explicit permission to stop (reduces all-or-nothing thinking)

Visible before starting (pre-emptive shame management)

Reinforced repeatedly (not just once)

This single line is the difference between:

✅ "I did one step, that's progress"

❌ "I only did one step, I failed"

Solution: Promote Permission to Primary UI

Option A: Always-Visible Permission Banner

text
┌─────────────────────────────────────┐
│ ✨ You can stop after any step.     │
│    Progress = valid.                │
├─────────────────────────────────────┤
│ Current bite:                       │
│ making a list                  5min │
│                                     │
│ [Mark Done]                         │
└─────────────────────────────────────┘
Option B: Contextual Permission on "Mark Done" Button

text
┌─────────────────────────────────────┐
│ Current bite:                       │
│ making a list                  5min │
│                                     │
│ ┌─────────────────────────────────┐ │
│ │   ✓ Mark Done & Stop Here       │ │
│ │   (That still counts)           │ │
│ └─────────────────────────────────┘ │
│                                     │
│ [Continue to Next Bite]             │
└─────────────────────────────────────┘
Why This Works:

Permission is VISIBLE (not hidden)

Reinforced at decision point (when they're about to stop)

Reduces guilt BEFORE it happens

5. "Choose Your Week" Modal is Overwhelming
Problem (Screen 6 - Weekly Rooms selector):

You're showing ALL 4 Weekly Rooms at once:

Gentle Week

Build Week

Repair Week

Chaos Week

With descriptions, that's 4 concepts × 2 lines each = 8 lines of text to parse.

Neurodivergent Issue:

Too many choices = decision paralysis

No default = "I don't know which I am" anxiety

Modal blocks progress (forced decision before starting)

UX Violation:
Forcing a framework decision before users understand the framework.

Solution: Progressive Onboarding

First-Time Users (Day 1):
Skip the Weekly Rooms entirely. Just let them use the system.

Day 3 Prompt:

text
┌─────────────────────────────────────┐
│ You've completed 5 bites this week! │
│                                     │
│ Want to match your system to your   │
│ energy? Try picking a "Weekly Room" │
│                                     │
│ [Show Me How]  [Not Yet]            │
└─────────────────────────────────────┘
When They Choose "Show Me How":

text
┌─────────────────────────────────────┐
│ This week feels like...             │
├─────────────────────────────────────┤
│ 🔥 Survival mode                    │
│    Everything's on fire             │
│                                     │
│ 🌙 Gentle mode                      │
│    Tired and need low demands       │
│                                     │
│ 🔨 Build mode                       │
│    Ready to move things forward     │
│                                     │
│ 🧹 Repair mode                      │
│    Catching up on what fell through │
│                                     │
│ [Skip This Week]                    │
└─────────────────────────────────────┘
Key Changes:

✅ Plain language ("Survival mode" not "Chaos Week")

✅ Short descriptions (one line max)

✅ Skip option (no forced choice)

✅ Introduced AFTER users understand basic flow

🟡 MEDIUM PRIORITY ISSUES
6. "All Your Bites" List Lacks Hierarchy
Problem (Screen 1):

The bite list shows:

text
☐ making a list          5 min
☐ driving to the store  10 min
But there's no visual separation between:

Current bite (what to do NOW)

Future bites (what comes AFTER)

Neurodivergent Issue:
ADHD brains need clear "what to do next" signals. Showing everything at once = visual noise.

Solution: Collapse Future Bites

text
┌─────────────────────────────────────┐
│ Current bite:                       │
│ ┌─────────────────────────────────┐ │
│ │ ☐ making a list          5 min  │ │
│ │                                 │ │
│ │ [Mark Done]                     │ │
│ └─────────────────────────────────┘ │
│                                     │
│ Next up (2 more bites)              │
│ [Show All]                          │
└─────────────────────────────────────┘
Why This Works:

Focuses attention on ONE thing

Reduces overwhelm (you don't see the whole list)

Progressive disclosure (show more on demand)

7. "Mark Done" Button Needs Celebration
Problem (Screen 1):

When you tap "Mark Done," what happens?

Silent transition to next bite?

Subtle checkmark animation?

Nothing?

Neurodivergent Issue:
ADHD brains need:

Immediate dopamine hit (visual reward)

Explicit acknowledgment (you DID something)

Micro-celebration (builds momentum)

Solution: Bite Completion Micro-Celebration

text
After tapping "Mark Done":

┌─────────────────────────────────────┐
│                                     │
│         ✨                          │
│     You did a thing!                │
│                                     │
│  [Keep Going]   [Stop Here]         │
│                                     │
│ "You're allowed to stop.            │
│  This already counts."              │
└─────────────────────────────────────┘

(Auto-advances after 2 seconds if no tap)
Why This Works:

Dopamine reward (emoji + affirmation)

Choice point (keep going OR stop)

Permission reinforced (reduces guilt for stopping)

8. Settings Page is Too Dense
Problem (Screen 4 - "You" settings):

Three toggles:

Haptic feedback

Timer notifications

Energy check-in

Plus stats:

Total bites

Steps done

Completion %

Neurodivergent Issue:
Stats like "0% Completion" create:

Performance anxiety

Gamification pressure (if you show %, users feel judged)

Shame (especially early on when it's low)

Solution: Hide Stats Until Meaningful

Day 1-7:
Don't show stats. Just settings.

After 7 days:

text
┌─────────────────────────────────────┐
│ This week you...                    │
│                                     │
│ ✨ Completed 8 bites                │
│ 🔥 Showed up 5 days                 │
│ 💚 That's momentum                  │
└─────────────────────────────────────┘
Key Differences:

✅ Absolute numbers (not percentages)

✅ Celebration framing ("That's momentum")

✅ No failure state (hide if zero activity)

🟢 NICE-TO-HAVE IMPROVEMENTS
9. Add "I Don't Know" Option to Energy Selector
Sometimes users genuinely don't know their energy level.

Add this option:

text
[Low]  [Medium]  [High]  [Not Sure]
If "Not Sure" selected:
→ Default to Medium routing
→ "We'll show medium-sized first steps. You can adjust anytime."

✅ Voice is perfect ("You showed up. That's what matters.")

❌ Permission statement is buried (promote to primary UI)

❌ Missing micro-celebrations (no dopamine reward for completion)
❌ Stats create pressure (hide until meaningful volume)

You're Solving the Right Problems:
✅ Task paralysis (Break It Down)
✅ Object permanence (Resume feature)
✅ All-or-nothing thinking (permission to stop)
✅ Energy blindness (Low/Medium/High routing)
✅ Shame spirals (celebration + validation)

But You're Introducing New Friction:
❌ Too many decisions at once (Break It Down modal)
❌ Visible failure states (0 of X)
❌ Abstract assessments (energy without context)
❌ Forced framework choices (Weekly Rooms too early)

💡 The Core Issue (In One Sentence)
You're trying to prevent overwhelm, but some of your UI patterns CREATE the overwhelm you're trying to solve.

Solution: More progressive disclosure, fewer upfront decisions, more post-action celebration.
